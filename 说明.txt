PiBot V3.28 Isolated Web Serial Firmware & Test Platform
========================================================

Project Overview
----------------
This is a static Web Serial platform for the PiBot V3.28 Isolated controller board. It provides firmware download, direct Web Serial flashing, a serial terminal, and hardware input/output testing.

Customers only need to open the HTTPS web page. They do not need Python, a local proxy, Arduino IDE, or XLoader.

Language versions:
- index.html: English homepage for international customers
- zh-cn.html: Chinese auxiliary page for internal use or Chinese customers

For future German, French, or other versions, copy index.html to de.html, fr.html, etc. The JavaScript runtime messages already switch between English and Chinese based on the page language and can be extended for more languages later.

Customer Requirements
---------------------
- CH340 / USB serial driver
- Chrome or Edge browser

CH340 driver download:
- Official download: https://www.wch-ic.com/downloads/CH341SER.ZIP.html
- Local download: CH341SER.zip in the website root directory

Server Requirements
-------------------
HTTPS is required.

Web Serial only works in secure browser contexts:
- HTTPS websites
- localhost for local testing

If the customer opens the page through plain http://, the browser will not allow serial port access.

Recommended Browsers
--------------------
- Chrome
- Edge

Not recommended:
- Firefox
- Safari
- Mobile browsers

Customer Workflow
-----------------
1. Open the HTTPS website in Chrome or Edge.
2. If the serial port is not listed, install the CH340 driver first:
   - Official download: https://www.wch-ic.com/downloads/CH341SER.ZIP.html
   - Local download: CH341SER.zip from this website
3. Connect the PiBot V3.28 Isolated board through USB.
4. Click "Connect Device".
5. Select the CH340 / COM port in the browser serial port picker.
6. After connection, use firmware flashing, hardware testing, and serial terminal commands as needed.

Web Page Features
-----------------
- Download official GRBL v1.1h firmware
- Download PiBot V3.28 Isolated custom firmware
- Flash ATmega328P bootloader directly through Web Serial
- Serial connection and GRBL command terminal
- Monitor XYZ limit, probe, abort, hold, and resume inputs
- Test M3/M4/M5 spindle outputs
- Test M7(A4), M8(A3), and M9 coolant outputs
- Test XYZ EN / DIR / CLK signal indicators

Serial Terminal Notes
---------------------
The page silently sends `?` in the background to refresh coordinates, limit switches, and output states. Automatic `<Idle|MPos...>` reports are not printed continuously in the terminal.

The terminal only shows MPos status when the customer clicks "Status" or manually enters `?`.

Firmware Flashing Notes
-----------------------
The page uses the browser Web Serial API to access the customer's local serial port and writes the ATmega328P bootloader with the stk500v1 protocol.

Common bootloader baud rates:
- 115200: common for Arduino Uno / Optiboot
- 57600: used by some older Nano bootloaders

Before flashing, close Arduino IDE, XLoader, serial monitor tools, and any other software that may occupy the COM port.

After flashing, follow the page prompt to disconnect and reconnect the serial port before testing.

Firmware Configuration Notes
----------------------------
Official GRBL 1.1h does not enable M7 by default.

The PiBot V3.28 Isolated custom firmware uses:
- D13 = Spindle Enable, active high
- D11 = Spindle PWM
- A4 = Coolant Mist (M7), active high
- A3 = Coolant Flood (M8), active high

The GRBL source file `grbl/config.h` must enable both:

#define ENABLE_M7
#define USE_SPINDLE_DIR_AS_ENABLE_PIN

Enabling only `USE_SPINDLE_DIR_AS_ENABLE_PIN` is not enough. M7/A4 will not work unless `ENABLE_M7` is also enabled.

Input Test Notes
----------------
The monitored inputs are:
- X/Y/Z limit switches
- Probe
- Abort / Reset
- Hold
- Resume

Inputs are active low. To test, short the corresponding Signal pin to GND with a jumper. The page status indicator should change immediately.

Output Test Notes
-----------------
Spindle outputs:
- M3 S1000: clockwise full output
- M3 S500: clockwise half PWM output
- M4 S1000: counterclockwise full output
- M4 S500: counterclockwise half PWM output
- M5: spindle off

Coolant outputs:
- M7: A4 high output
- M8: A3 high output
- M9: turn off M7/M8
